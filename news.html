September 4, 2016: AI Algorithm

In my time programming, I've had to write many algorithms of varying complexity. However, while working on "A Pirate's Life," I came across one of the most challenging obstacles I've yet faced. Making a challenging, yet not impossible to beat enemy AI.

I think I've come close. It's not perfect, but it's near fully functional, and is the best solution I've come up with out of the many I've tried so far.

The algorithm in pseudocode is the following running in a loop:

  1. Assign an enemy to a ship
  2. If (Distance to enemy > Cannon Range):
    • Chase
    Else:
    • Attack
  1. Chase
    1. Calculate the angle to the target. (The picture to the left describes what I mean).
    2. If (angle to target < 10):
      • Turn Right
      Else:
      • Turn Left
  2. Attack
    • If (distance to target < Cannon Range/2 ):
      • Line up cannons
        • The reason cannon range is divided by two, is because the ship must turn to face the enemy. So if the ship turned as soon as cannons were in range, by the time it lined up, the enemy would be out of range again.
    • Else:
      • Return to chasing
  3. Lining up the Cannons
    • If (angle to target < 90 Degrees):
      • Turn Right
    • Else If (angle to target >= 90 Degrees):
      • Turn Left
    • If (angle to target > 85 Degrees OR angle to target < 95 Degrees) (e.g. the target is between 85 and 95 degrees)
      • If (target is on left): Shoot left
        If (target is on right): Shoot right, same for forward and back.
        Which side the enemy is on is determined by a complex algorithm that I will go over in another post.
